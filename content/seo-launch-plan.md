## ДЕНЬ 1 — Верификация в GSC + Яндекс.Вебмастер

### Шаг 1.1: Верификация в Google Search Console

**Рекомендуемый метод: HTML meta-тег** (лучший вариант для Next.js на Vercel — не требует правки DNS, работает сразу).

1. Открыть [Google Search Console](https://search.google.com/search-console)
2. Нажать «Добавить ресурс» → выбрать «URL-префикс»
3. Ввести URL сайта: `https://[your-vercel-url]/`
4. Выбрать метод «HTML-тег»
5. Скопировать код вида:
   ```
   <meta name="google-site-verification" content="XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX" />
   ```
6. Добавить в `src/app/layout.tsx`:

```tsx
export const metadata: Metadata = {
  // ... существующие поля ...
  verification: {
    google: "XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX",
  },
};
```

Next.js автоматически рендерит `<meta name="google-site-verification" ...>` в `<head>`.

7. Задеплоить изменения (push в main → Vercel авто-деплой)
8. В GSC нажать «Подтвердить» — обычно срабатывает в течение 1–2 минут

**Альтернативный метод: HTML-файл** (если meta-тег не работает):

1. Скачать файл верификации вида `googleXXXXXXXX.html`
2. Положить в `/public/googleXXXXXXXX.html`
3. Задеплоить → файл будет доступен по `https://[url]/googleXXXXXXXX.html`
4. Нажать «Подтвердить» в GSC

---

### Шаг 1.2: Верификация в Яндекс.Вебмастер

**Рекомендуемый метод: HTML meta-тег** (аналогично GSC).

1. Открыть [Яндекс.Вебмастер](https://webmaster.yandex.ru)
2. Нажать «+» → ввести URL сайта
3. Выбрать «Meta-тег»
4. Скопировать код:
   ```
   <meta name="yandex-verification" content="XXXXXXXXXXXXXXXX" />
   ```
5. Добавить в `src/app/layout.tsx` (в блок `other` или как отдельный тег через `<head>`):

```tsx
// Вариант 1 — через metadata.other
export const metadata: Metadata = {
  // ... существующие поля ...
  other: {
    "yandex-verification": "XXXXXXXXXXXXXXXX",
  },
};
```

```tsx
// Вариант 2 — через отдельный компонент в layout.tsx
// В <head> добавить:
<meta name="yandex-verification" content="XXXXXXXXXXXXXXXX" />
```

6. Задеплоить → нажать «Проверить»

**Важно:** Яндекс может занять до 24 часов для подтверждения — это нормально.

## ДНИ 1–3 — Первичная индексация

### Шаг 2.1: Обновить robots.txt с реальным URL

Файл `src/app/robots.ts` сейчас содержит плейсхолдер. После деплоя:

```ts
// src/app/robots.ts — обновить URL
import type { MetadataRoute } from "next";

export default function robots(): MetadataRoute.Robots {
  return {
    rules: {
      userAgent: "*",
      allow: "/",
    },
    sitemap: "https://[REAL-URL]/sitemap.xml",
  };
}
```

Проверить доступность: `https://[url]/robots.txt` — должен вернуть правильный контент.

---

### Шаг 2.2: Сабмит sitemap.xml в GSC

1. В GSC → «Файлы Sitemap» (левое меню)
2. Нажать «Добавить файл Sitemap»
3. Ввести: `sitemap.xml`
4. Нажать «Отправить»

Ожидаемый результат: статус «Успешно», количество обнаруженных URL.

**Проверка sitemap:** открыть `https://[url]/sitemap.xml` — должны быть все ключевые страницы:
- `/` (главная)
- `/about/`
- `/services/` и каждая услуга
- `/portfolio/`
- `/contact/`
- `/blog/` + все статьи

---

### Шаг 2.3: Сабмит sitemap.xml в Яндекс.Вебмастер

1. Яндекс.Вебмастер → «Индексирование» → «Файлы Sitemap»
2. Нажать «Добавить Sitemap»
3. Ввести полный URL: `https://[url]/sitemap.xml`
4. Нажать «Добавить»

---

### Шаг 2.4: URL-инспекция ключевых страниц в GSC

Для каждой страницы ниже: GSC → «Инспекция URL» → вставить URL → «Запросить индексирование»:

| Приоритет | URL | Причина важности |
|-----------|-----|------------------|
| 🔴 Критично | `/` | Главная, max трафик |
| 🔴 Критично | `/services/` | Ключевые конверсии |
| 🔴 Критично | `/contact/` | Целевая страница |
| 🟡 Высокий | `/about/` | E-E-A-T сигналы |
| 🟡 Высокий | `/blog/` | Индекс блога |
| 🟢 Средний | Каждая страница услуги (8 штук) | Семантика |
| 🟢 Средний | `/portfolio/` | Trust сигналы |

Лимит GSC: 10 запросов в день. Делайте по приоритету.

---

### Шаг 2.5: Проверка Coverage Report в GSC

Через 24–48 часов после сабмита:

1. GSC → «Индексирование» → «Страницы»
2. Смотреть вкладки:
   - **«Проиндексировано»** — ожидаем рост количества страниц
   - **«Исключено»** — проверить, нет ли важных страниц
   - **«Ошибки»** — исправить немедленно

Частые ошибки для Next.js на Vercel:
- `Страница перенаправляет на другую` — проверить canonical и редиректы
- `Не найдено (404)` — проверить sitemap на опечатки
- `Обнаружено — индексирование не предусмотрено` — проверить robots.txt и noindex

---

### Шаг 2.6: Проверка в Яндекс.Вебмастер

1. Яндекс.Вебмастер → «Индексирование» → «Страницы в поиске»
2. Через 2–5 дней должны появиться первые проиндексированные страницы
3. Раздел «Ошибки» — исправить критические

**Инструмент «Проверка URL»** (Яндекс): аналог GSC URL Inspection — проверить главную и страницы услуг вручную.

### Красные флаги: что требует немедленного действия

| Ситуация | Что делать |
|----------|-----------|
| После 7 дней 0 страниц в GSC «Проиндексировано» | Проверить robots.txt — нет ли `Disallow: /`. Проверить noindex в коде |
| После 14 дней 0 impressions в GSC | Проверить верификацию (переделать). Проверить, что сабмит sitemap прошёл успешно |
| Ошибка «Страница с noindex» на важных страницах | Убрать `<meta name="robots" content="noindex">` если есть |
| Ошибка «Redirect» на главной | Проверить canonical — не должно быть `www.` редиректов |
| PageSpeed < 50 на мобильном | Оптимизировать изображения (WebP), проверить LCP элемент |
| В Яндекс нет страниц через 10 дней | Добавить сайт в Яндекс.Вебмастер повторно + проверить HTTP/HTTPS |
| GSC показывает «Обнаружено – индексирование не предусмотрено» | Проверить robots.txt, убедиться что Allow: / работает |
