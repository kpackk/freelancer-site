Если вы продаёте на Wildberries и Ozon, то знаете главную боль мультиканальных продаж: остатки «плывут», цены нужно обновлять вручную на каждой площадке, заказы разбросаны по разным кабинетам.

Интеграция через API решает это: данные синхронизируются автоматически, исключая ручной труд и ошибки. В этой статье разберём, как работает интеграция, что она даёт бизнесу и сколько стоит.

---

## Зачем интегрировать Wildberries и Ozon через API

### Проблема без интеграции

Типичная ситуация: у вас 500 SKU, продажи идут на WB, Ozon и собственном сайте. Чтобы обновить цену — заходите в три кабинета, меняете вручную. Обновление остатков — то же самое.

В пиковые периоды (11.11, Новый год) это не просто неудобство — это прямые потери: пересорт, возвраты за «нет в наличии», штрафы за отмены.

### Что даёт интеграция

- **Единое управление остатками:** обновили склад в 1С или своей системе → изменения автоматически ушли на WB и Ozon
- **Синхронизация цен:** изменили прайс в одном месте → все площадки обновились
- **Централизованные заказы:** заказы со всех каналов приходят в одну систему
- **Аналитика в одном месте:** продажи WB + Ozon + сайт = единый дашборд

## Как работает Wildberries API

Wildberries предоставляет официальный REST API (`openapi.wb.ru`) с доступом через API-ключ из личного кабинета.

**Основные возможности:**
- Управление товарами: создание карточек, обновление описаний и фото
- Управление ценами: обновление цен и скидок
- Склады и остатки: обновление FBS-остатков по складу
- Заказы и сборочные задания: получение новых заказов, статусы
- Аналитика: статистика продаж, ABC-анализ, воронка

**Ограничения:**
- Некоторые методы имеют rate limits (количество запросов в минуту)
- FBW-остатки обновляются через поставки, не через API
- Контент (карточки товаров) обновляется с задержкой до 24 часов

## Как работает Ozon API

Ozon Seller API (`api-seller.ozon.ru`) — более зрелый и документированный, чем WB.

**Основные возможности:**
- Товары: создание и редактирование карточек, атрибуты, фото
- Цены и акции: массовое обновление цен, участие в акциях
- Склады FBS: управление остатками
- Заказы: получение заказов, генерация этикеток
- Аналитика: отчёты по продажам, возвратам, трафику

**Плюс Ozon API:** более подробная документация, стабильный rate limit, sandbox для тестирования.

## Что обычно интегрируют

### Синхронизация остатков (самый частый запрос)

Схема: ваша учётная система (1С, ERP, Google Sheets, собственная база) → Python-скрипт → Wildberries API + Ozon API.

Обновление происходит по расписанию (каждые 15–30 минут) или по триггеру (продажа на одной из площадок).

### Мультиканальный приём заказов

Все заказы с WB и Ozon автоматически создаются в вашей CRM (Bitrix24, amoCRM) или 1С. Менеджеры работают в одном интерфейсе.

### Автоматическое обновление цен

Репрайсер: скрипт мониторит конкурентов и автоматически корректирует ваши цены в заданном диапазоне (не ниже X, не выше Y).

### Выгрузка аналитики

Ежедневный/еженедельный отчёт: продажи по площадкам, топ товаров, ABC-анализ, динамика выкупа — в Excel или Google Sheets.

## Технологии разработки

Для интеграций использую:

- **Python** — основной язык (requests, httpx для API-вызовов)
- **PostgreSQL** — хранение данных о товарах, заказах, остатках
- **Redis** — кэширование и очереди задач
- **Celery** — асинхронные задачи и планировщик
- **FastAPI или Flask** — если нужен собственный API-слой

Деплой: Docker + VPS или serverless (для небольших интеграций).

## Реальные кейсы

### Синхронизация остатков WB + Ozon + сайт

**Ситуация:** магазин одежды, 800 SKU, три канала продаж. Ежедневно 3 оверсейла из-за рассинхрона остатков.

**Решение:** Python-скрипт читает остатки из 1С-выгрузки каждые 20 минут, обновляет WB FBS API, Ozon Stock API и REST API собственного сайта.

**Результат:** оверсейлы сократились до 0, штрафы WB прекратились, 2 часа ручного труда в день → 0.

**Стоимость:** 18 000 ₽, разработка 8 дней.

---

### Репрайсер для Ozon

**Ситуация:** продавец электроники хотел автоматически снижать цены при акциях конкурентов, но не ниже себестоимости + 15%.

**Решение:** скрипт мониторит топ-20 конкурентов по ключевым SKU через Ozon Price Analytics, корректирует цены с учётом минимальной маржи.

**Результат:** доля в корзине выросла на 23%, маржа не просела.

**Стоимость:** 25 000 ₽ (сложная логика + постоянный мониторинг).

---

### Мультиканальный приём заказов в Bitrix24

**Ситуация:** заказы с WB и Ozon менеджер вручную переносил в CRM. 40 заказов в день = 2 часа ручного труда.

**Решение:** webhook-интеграция: новые заказы с обеих площадок автоматически создают сделки в Bitrix24 с нужными полями.

**Результат:** 2 часа ежедневно → 0. Менеджеры увидели полную картину воронки.

**Стоимость:** 14 000 ₽, разработка 6 дней.
